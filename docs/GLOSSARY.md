# Глоссарий: трекинг, карты и приложение

Термины предметной области для личного трекера поездок (вело, пешие, бег).

---

## Карты и отображение

**Тайл (tile)** — небольшой растровый фрагмент карты (обычно 256×256 пикселей). Карта в приложениях собирается из множества тайлов: при зуме и сдвиге подгружаются новые тайлы. Тайлы запрашиваются с сервера (например, OpenStreetMap) по координатам и уровню масштаба (zoom level). *Офлайн-карты* — это кэширование уже загруженных тайлов на устройстве.

**OpenStreetMap (OSM)** — открытый проект карты мира, создаваемой сообществом. Данные и тайлы можно использовать бесплатно (с соблюдением лицензии). В приложениях часто подставляют OSM-тайлы вместо платных карт (Google Maps и т.п.).

**MapLibre** — открытый движок для отображения карт (форк Mapbox GL). Поддерживает векторные и растровые карты, маркеры, полилинии, офлайн. Используется, в частности, в expo-osm-sdk.

**Полилайн (polyline)** — линия на карте, заданная последовательностью точек (широта, долгота). В нашем контексте — отрисованный маршрут поездки по массиву записанных координат.

**Zoom level** — уровень масштаба карты (целое число, например 10–18). Чем больше число, тем крупнее масштаб (больше деталей, меньше охват).

## Геолокация и трекинг

**Геолокация** — определение текущего положения устройства (широта, долгота, точность, высота при наличии) по GPS, сотовой сети и Wi‑Fi.

**Координаты** — пара «широта (latitude), долгота (longitude)» в градусах. Используются для точки на карте и для записи маршрута.

**Трек (track)** — последовательность координат, записанная за время поездки. В приложении хранится как массив точек; при сохранении может быть экспортирован в GPX или JSON.

**Трекинг** — процесс непрерывной записи координат во время активности (поездка, прогулка, пробежка). В документации также: состояние *Tracking* — «идёт запись».

**Фоновая геолокация** — получение координат, когда приложение свёрнуто или экран выключен. На Android для этого нужен *foreground service* и разрешение *ACCESS_BACKGROUND_LOCATION*.

**Foreground service (сервис переднего плана)** — на Android: фоновый процесс, который считается «активным» и меньше убивается системой. Обязательно показывает постоянное уведомление. Используется для непрерывного трекинга при выключенном экране.

**ACCESS_BACKGROUND_LOCATION** — разрешение Android на получение геолокации в фоне. Запрашивается отдельно после разрешения на геолокацию «во время использования»; без него запись маршрута при свёрнутом приложении ненадёжна или невозможна.

## Расчёты и форматы

**Haversine (формула Хаверсина)** — формула для приближённого расчёта расстояния между двумя точками на сфере по их широте и долготе. Используется для подсчёта дистанции поездки по массиву координат (сумма расстояний между соседними точками). Достаточна для не очень длинных маршрутов; для точных расчётов на больших расстояниях иногда используют другие модели (например, Винсенти).

**GPX (GPS Exchange Format)** — открытый XML-формат для хранения треков, маршрутов и путевых точек. Поддерживается многими приложениями и сервисами (Strava, Garmin, навигаторы). Удобен для экспорта и импорта поездок.

**FIT** — проприетарный, но широко используемый формат Garmin для активностей (трек, пульс, мощность и т.д.). В текущем скоупе не обязателен.

**TCX (Training Center XML)** — XML-формат для тренировок, поддерживается Garmin и другими. Может использоваться для обмена данными наряду с GPX.

## Состояния и сценарии приложения

**Idle** — состояние «ожидание»: приложение готово начать запись, трекинг не идёт.

**Tracking** — состояние «запись»: идёт запись координат в массив маршрута, считаются дистанция и время.

**Finished** — состояние «завершено»: сессия остановлена, отображается итог (маршрут на карте, дистанция, время); поездку можно сохранить в историю.

**Сегмент (в контексте Strava)** — заранее заданный участок маршрута (например, подъём), по которому сравнивают время разных заездов и строят лидерборды. В нашем продукте вне скоупа.

**Лидерборд (leaderboard)** — таблица лидеров по сегменту или соревнованию. В нашем продукте вне скоупа (нет соцфункций).

## Платформа и стек

**Expo Go** — приложение для быстрого запуска проектов на Expo без сборки. Часть возможностей (например, свой foreground service, нативные модули вроде expo-osm-sdk) в Expo Go недоступна или ограничена.

**Development build (dev build)** — собственная сборка приложения (через EAS Build или `expo run:android`), в которую входят все нативные модули проекта. Нужна для фонового трекинга, кастомных карт (expo-osm-sdk) и т.п.

**AsyncStorage** — простое ключ–значение хранилище в Expo/React Native. Данные сохраняются локально на устройстве. Подходит для настроек и небольшой истории; для очень больших объёмов точек могут использоваться файлы или SQLite.

**expo-location** — модуль Expo для доступа к геолокации: разовый запрос координат, подписка на обновления, фоновые задачи (TaskManager + startLocationUpdatesAsync) с поддержкой foreground service на Android.

## Разное

**Поездка / активность** — одна записанная сессия: массив координат, время начала и окончания, посчитанная дистанция (и при желании имя, заметки). Сохраняется в истории.

**Маршрут** — в контексте приложения то же, что трек: последовательность точек, образующая линию на карте.

**Путевая точка (waypoint)** — отдельно отмеченная точка на карте (например, старт, финиш, достопримечательность). В базовом скоупе может не использоваться; в GPX поддерживается.
