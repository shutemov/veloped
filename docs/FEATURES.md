# Фичи: реализуемость открытым стеком на Android

Всё перечисленное ниже реализуемо открытым ПО на Android без платных API: карты — OSM, геолокация — Expo, хранение — локальное.

---

## 2.1. Ядро (трекинг и карта)

| Фича | Реализуемо | Стек / библиотеки | Примечание |
|------|------------|-------------------|------------|
| Отображение карты | Да | expo-osm-sdk (MapLibre + OSM) или react-native-maps с OSM-тайлами | expo-osm-sdk требует development build (не Expo Go). Без API-ключей. |
| Трекинг в реальном времени | Да с оговорками | expo-location: `startLocationUpdatesAsync` + foreground service | Запуск foreground service только из foreground; известные баги при уходе в фон. |
| Отрисовка маршрута (Polyline) | Да | expo-osm-sdk, react-native-maps | Поддержка полилиний из коробки, в т.ч. с OSM. |
| Дистанция и время в пути | Да | Клиентский расчёт: Haversine по массиву координат, таймер сессии | Без облака. |
| Состояния Idle / Tracking / Finished | Да | React state или context | Реализация на уровне состояния приложения. |

Карта, запись трека, полилайн, дистанция и время полностью реализуемы открытым стеком на Android.

---

## 2.2. Хранение и история

| Фича | Реализуемо | Стек / библиотеки | Примечание |
|------|------------|-------------------|------------|
| Локальное хранение поездок | Да | AsyncStorage (Expo), при необходимости expo-secure-store; для больших объёмов — expo-sqlite или файлы (JSON/GPX) | AsyncStorage достаточен для старта; SQLite/файлы — при росте числа точек. |
| История поездок | Да | Данные из AsyncStorage/SQLite/файлов | Список сохранённых треков, экран детали с картой и метриками — всё на клиенте. |
| Периодическое сохранение во время трекинга | Да | Запись в AsyncStorage или файл по таймеру / по N новым точкам | Снижает потерю трека при закрытии или падении приложения. |

## 2.3. Экспорт и форматы

| Фича | Реализуемо | Стек / библиотеки | Примечание |
|------|------------|-------------------|------------|
| Экспорт в GPX | Да | react-native-gpx (генерация) или чистый JS | Совместимо с другими приложениями и сервисами. |
| Импорт GPX | Да | react-native-gpx или @we-gold/gpxjs | Парсинг GPX, отображение маршрута на карте через Polyline. |

---

## 2.4. Дополнительные фичи

| Фича | Реализуемо | Стек / библиотеки | Примечание |
|------|------------|-------------------|------------|
| Офлайн-карты | Да | Кэширование тайлов OSM (react-native-maps или expo-osm-sdk/MapLibre) | Нужен лимит по объёму хранилища. |
| Пауза / продолжение поездки | Да | Логика состояния приложения | Не добавлять точки в маршрут во время паузы; при желании сохранять сегменты. |
| Базовая статистика | Да | Расчёт по сохранённым трекам на клиенте | Недельная/месячная дистанция, количество поездок. |
| Уведомление во время трекинга (Android) | Да | Foreground service в expo-location | Постоянное уведомление; текст можно кастомизировать (дистанция, время). |
| Разрешения (в т.ч. фоновая геолокация) | Да | expo-location, настройка app.json и infoPlist | Android: ACCESS_BACKGROUND_LOCATION; iOS: UIBackgroundModes, NSLocation* описания. |

---

## 2.5. Вне текущего скоупа (упомянуть, не как «реализуемо без оговорок»)

| Тема | Реализуемо | Примечание |
|------|------------|------------|
| Облачная синхронизация | Да, но не «только OSS на устройстве» | Нужен свой или сторонний бэкенд. Указать в продукте как возможное расширение. |
| Социальные функции, лидерборды, сегменты Strava | Вне скоупа | Не входят в «личное использование». |
| Интеграция с Wear OS / часами | Возможное развитие | Отдельная тема; в первом документе фич — без детализации стека. |
